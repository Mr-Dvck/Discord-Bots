Dyno Feature Atlas

This map reorganizes the Dyno dashboard into capability zones, then attaches the command layer so you can see which features are mostly dashboard-configured, which are user-facing, and which are staff-operational.

Core/System Standard Premium Command-driven

How to read it

Modules are grouped by intent instead of dashboard order. Each card shows the operating model, likely actors, and command surface where docs expose one.

Control surfaces

  • Dashboard-first: setup lives in Dyno web UI.
  • Runtime commands: members or staff invoke commands in Discord.
  • Hybrid: configured in dashboard, consumed by commands.

Command spine

Dyno uses a changeable prefix and defaults to ?. The server dashboard command tab can enable or disable commands and restrict them by allowed or ignored roles and channels.

Capability zones

Safety

Moderation stack

Modules: Moderation, Automod, Action Log, Autoban, Slowmode, Auto Delete, Auto Purge.

Works as detect → rate-limit/delete/punish → log. This is the server governance layer.

Commands: moderation commands are unlocked by the Moderation module; diagnostics and module toggling also exist through ?diagnose, ?module, and ?modules.

Onboarding

Identity & entry

Modules: Welcome, Announcements, Autoroles, Reaction Roles, Forms.

Works as join event → greeting / announcement → role assignment → optional intake workflow.

Commands: Autoroles exposes joinable rank commands such as ?rank, ?ranks, ?addrank, and ?delrank.

Automation

Message automation

Modules: Auto Message, Autoresponder, Message Embedder, Custom Commands, Tags.

Works as timer or text trigger → generate response / embed / reusable snippet / command macro.

Commands: Custom Commands can run regular Dyno commands and variables; Tags is user or role mediated content recall.

Engagement

Retention loop

Modules: Levels, Giveaways, Starboard, AFK, Reminders, Highlights, Fun, Economy.

Works as activity → reward / recognition / reminder / entertainment. This is your community flywheel.

Commands: Fun docs expose commands like ?poll, ?roll, ?rps, ?github, and media lookups.

Support

Staff operations

Modules: Tickets, Forms, Action Log.

Works as request intake → private channel / transcript → staff response → archival trace.

Dashboard-heavy, with staff actions centered in configured panels rather than casual public commands.

Creator feeds

External notifications

Modules: Reddit, Twitch, YouTube, TikTok, Kick.

Works as external publish/live event → Dyno notification post. Good for creators, stream teams, launches, and brand channels.

Reddit is standard on the page; Twitch, YouTube, TikTok, and Kick are marked Premium on the dashboard.

Command cartography

Core control

Platform commands

?module toggles a module on or off, ?modules lists available modules and their enabled state, and ?diagnose checks a command or module for problems.

Use these as your debugging and systems-admin layer.

Role control

Role management

?role add, ?role remove, ?role toggle, ?role removeall, ?role all, ?role humans, ?role in, ?role status, and ?role cancel form a substantial role-ops toolkit.

This sits between moderation and onboarding because it can assign identity, clean up groups, and do mass operations.

Joinable ranks

Member self-service

?rank and ?ranks are the member-facing rank surface, while ?addrank and ?delrank are admin controls for creating those joinable roles.

This makes Autoroles a hybrid system: configured in dashboard, consumed in chat.

Fun/social

Engagement commands

Examples documented by Dyno include ?cat, ?dadjoke, ?dog, ?flip, ?poll, ?roll, ?rps, ?pokemon, and ?github.

These are low-risk, high-frequency commands that make the bot feel alive to members.

Operational path

1. Configure
Dashboard modules
Enable a module, set channels, roles, triggers, schedules, or permissions.
2. Govern
Commands tab
Allow or ignore commands by role or channel and disable unused surfaces.
3. Operate
Discord runtime
Members and staff trigger ranks, moderation, fun, reminders, tags, and custom command flows.
4. Observe
Logs & actions
Action Log and mod logging provide the audit trail for what happened.
